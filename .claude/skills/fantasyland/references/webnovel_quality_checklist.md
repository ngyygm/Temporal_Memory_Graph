# 网文质量检查清单（Fantasyland 记忆增强版）

## 一致性检查（Fantasyland 核心）

### 角色属性一致性

- [ ] 角色外貌描写和设定一致（疤痕、发色、身高、体型等）
- [ ] 角色性格表现和设定一致（不突然变性格）
- [ ] 角色能力和当前等级匹配
- [ ] 角色说话方式和设定一致

**检查方法：** `quick_search(query="[角色名]的外貌/性格/能力设定")`

### 角色状态连续性

- [ ] 上一章的伤势/疲劳状态在延续
- [ ] 角色拥有的道具/装备没有凭空出现或消失
- [ ] 角色当前位置和上章结尾衔接
- [ ] 角色情绪状态变化有铺垫

**检查方法：** `ask(question="[角色名]上一章结束时的完整状态")`

### 世界观一致性

- [ ] 力量体系规则没有违反（升级条件、限制、代价）
- [ ] 地理描写和设定一致（距离、方向、地形）
- [ ] 社会/组织规则没有违反
- [ ] 时间流逝合理（不会白天突然变昨晚）

**检查方法：** `quick_search(query="[相关世界观设定]")`

### 因果逻辑

- [ ] 角色做决定的动机在前文有铺垫
- [ ] 角色获取信息有合理路径（不凭空知道）
- [ ] 事件发展有因果链
- [ ] 没有信息泄露（读者知道但角色不知道的，角色没有表现出知道）

**检查方法：** `ask(question="[角色名]是否有可能知道[某信息]？他的信息来源是什么？")`

### 伏笔状态

- [ ] 本章计划操作的伏笔都已处理
- [ ] 未回收伏笔没有遗忘
- [ ] 新埋伏笔已记录
- [ ] 伏笔回收有铺垫痕迹

**检查方法：** `quick_search(query="所有未回收的伏笔")`

---

## 章内结构

- [ ] 本章开头前几段有抓手，不是平铺日常
- [ ] 本章中段至少推进剧情、关系、设定理解中的一项
- [ ] 到后段时，局面已经和开头不同
- [ ] 章末停在变化发生的那一拍

## 开头检查

- [ ] 前 300-500 字有抓手
- [ ] 不是天气/日常/背景介绍开场
- [ ] 主角尽快出场
- [ ] 卖点或危机尽快显形

## 黄金三章检查

- [ ] 第 1 章有明确事件
- [ ] 第 2 章冲突升级或卖点展开
- [ ] 第 3 章建立继续追读的理由

## 本章推进检查

- [ ] 本章有核心事件
- [ ] 本章局面发生变化
- [ ] 本章不是纯过渡水章
- [ ] 至少推进剧情、关系、设定理解中的一项

## 信息投放检查

- [ ] 没有大段背景直灌
- [ ] 信息尽量跟着冲突出现
- [ ] 读者不需要一口气记住太多设定

## 场景检查

- [ ] 每场戏有目标
- [ ] 每场戏有阻碍
- [ ] 每场戏有变化
- [ ] 场景里有人在做事，不是原地说感受
- [ ] 没有可以整段删除却不影响剧情的部分

## 章末检查

- [ ] 章末停在变化上
- [ ] 章末有危机/决定/发现/反转之一
- [ ] 章末不是总结式收尾
- [ ] 章末能自然拉到下一章

## 语言检查

- [ ] 没有空泛抒情堆砌
- [ ] 没有连续大段同义情绪重复
- [ ] 句子顺滑，读起来不卡
- [ ] 对话符合人物身份
- [ ] 关键情绪尽量用动作、反应、对白落地
- [ ] 没有典型 AI 味（"空气凝固"、"五味杂陈"、"深邃眸子"等）

## 容易水的地方

- [ ] 没有大段背景解释
- [ ] 没有角色重复表达同一个情绪
- [ ] 没有连续几段都不给新信息或新变化
- [ ] 没把本章写成只过桥、不落事的过渡章

## 连载检查

- [ ] 没忘掉前文已经给出的承诺
- [ ] 没突然引入过多新设定
- [ ] 伏笔有推进，不是只埋不收
- [ ] 当前剧情仍然服务于故事引擎

## 记忆写入检查（Fantasyland 专属）

- [ ] 每段写前都执行了 `quick_search`
- [ ] 每段写后都执行了一致性校验
- [ ] 每段都 `remember` 存入了记忆
- [ ] 完整章节已 `remember` 存入
- [ ] 状态变更已 `remember` 记录
- [ ] 关键角色 `evolve_entity_summary` 已更新
- [ ] 已 `dream_run` 发现新关联
- [ ] 伏笔操作已记录在记忆中
