### 问题1：MongoDB 事务超时

迁移脚本在处理大批量数据时报了 `MongoError: TransactionExceededTimeLimit (code 257)`。查了一下，MongoDB 事务默认超时 60 秒，我的 batch 有 10000 条，事务里还嵌套了索引检查。

解决方案：把大 batch 拆成 500 条一组，每组单独事务。代码改了一行：

```python
# Before
batch_size = 10000

# After
batch_size = 500  # 每个事务不超过 60s
```

参考文档：https://www.mongodb.com/docs/v7.0/core/transactions/

### 问题2：Redis 连接池泄露

跑完迁移后 Grafana 告警 `redis-cluster-01.internal:6379` 连接数飙到 800（正常 200 左右）。原因是 `aioredis` 的连接池没有正确关闭。

修复：
```python
# Before — 每次都创建新连接
async def get_redis():
    return await aioredis.from_url("redis://redis-cluster-01:6379")

# After — 复用连接池
_redis_pool = None
async def get_redis():
    global _redis_pool
    if _redis_pool is None:
        _redis_pool = await aioredis.from_url(
            "redis://redis-cluster-01:6379",
            max_connections=50,
            decode_responses=True
        )
    return _redis_pool
```

部署到 k8s 后验证：`kubectl logs -n data-pipeline deployment/data-pipeline-api | grep redis` 连接数恢复正常。
