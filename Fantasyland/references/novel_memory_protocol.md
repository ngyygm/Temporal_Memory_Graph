# Fantasyland 记忆交互协议

这份文件定义了 Fantasyland 在写作过程中与 Deep-Dream 的所有交互规则。

## source_name 命名规范

统一的命名前缀让后续检索更精准：

| 前缀 | 用途 | 示例 |
|------|------|------|
| `设定-*` | 世界观设定 | `设定-世界观`, `设定-角色-林嘿嘿`, `设定-力量体系` |
| `大纲-*` | 故事大纲 | `大纲-总纲`, `大纲-第一卷`, `大纲-第3章` |
| `预设-*` | 章节场景预设 | `预设-第3章` |
| `正文-*` | 正式正文 | `正文-第3章-深渊之下`, `正文-第3章-P2` |
| `状态-*` | 章节结束状态快照 | `状态-第3章结束` |
| `伏笔-*` | 伏笔记录 | `伏笔-001-暗河回声` |
| `修改-*` | 设定变更记录 | `修改-角色-林嘿嘿-突破七品` |

## 调用时机与频率

### 创作启动时（一次性）

```
graph_summary          → 确认图谱状态
```

### 世界观构建时（每类设定）

```
remember               → 写入设定（wait=true）
quick_search           → 验证抽取是否完整
ask                    → 确认关键信息是否存对
create_entity          → 补全遗漏实体
create_relation        → 补全遗漏关系
```

### 大纲规划时（每层大纲）

```
quick_search           → 读取已有设定
remember               → 存储大纲
dream_run              → 发现隐藏关联
```

### 章节开始前（每章一次）

```
quick_search           → 读取本章大纲
ask                    → 确认前文状态
quick_search           → 检查未回收伏笔
batch_profiles         → 获取出场角色画像
remember               → 存储章节场景预设
```

### 段落写作时（每段重复）

```
# 写前（必须）
quick_search           → 召回场景相关记忆
ask                    → 确认角色当前状态

# 写后（必须）
ask                    → 一致性校验
remember               → 存储写好的段落
```

### 章节完成时（每章一次）

```
remember               → 存储完整章节
remember               → 记录状态变更
evolve_entity_summary  → 更新关键角色状态
dream_run              → 发现新关联
```

## 调用频率参考

| 阶段 | 每章调用次数 | 主要工具 |
|------|------------|---------|
| 章节预设 | 3-5 | quick_search, ask, batch_profiles, remember |
| 段落写作 | 每段 3-6 | quick_search, ask, remember |
| 章节收尾 | 3-5 | remember, dream_run, evolve_entity_summary |
| **每章总计** | **约 20-40** | |

## 记忆内容规范

### 角色设定写入模板

```
角色：[名字]
- 身份/职业：[具体身份]
- 年龄：[数字]
- 性格：[3-5个关键词，附简短说明]
- 外貌：[具体描写，含标志性特征，如疤痕、纹身、常穿戴]
- 背景：[出身、重要经历]
- 能力：[当前等级/技能]
- 弱点：[性格缺陷/恐惧/生理弱点]
- 动机：[最想要什么]
- 说话方式：[语气、常用口头禅、说话节奏]
```

### 章节状态变更写入模板

```
第X章结束状态变更：
- [角色A]：[物理状态变化，如受伤/恢复/获得道具]
           [心理状态变化，如信任/怀疑/决定]
           [关系变化，如与新角色建立联系]
- [角色B]：[同上]
- 新发现：[本章揭示的世界观信息]
- 未解之谜：[新的悬念]
- 伏笔操作：[埋设/铺垫/回收了什么]
```

### 伏笔写入模板

```
伏笔-[编号]：[伏笔名称]
- 类型：悬念 / 暗示 / 铺垫 / 反转线索
- 埋设章节：第X章
- 内容：[具体描述埋了什么]
- 铺垫进度：[已铺垫了几次，分别在哪章]
- 预计回收：第X-Y章
- 揭示方向：[实际真相是什么]
- 相关实体：[涉及的角色/物品/地点的 family_id]
```

## 一致性校验协议

### Layer 1: 实体属性校验（自动）

写到角色/物品/地点时：

```
quick_search(query="[实体名]的[属性类别]")
```

比对当前描述是否和记忆一致。不一致 → 修改段落。

### Layer 2: 状态连续性校验（自动）

写到角色行为时：

```
ask(question="[角色名]当前的身体状态、情绪状态、装备情况是什么？")
```

检查行为是否和当前状态匹配。

### Layer 3: 叙事逻辑校验（半自动）

写到关键情节转折时：

```
ask(question="""
检查以下情节的逻辑：
1. [角色名]是否有可能知道这个信息？（信息获取路径检查）
2. 从[事件A]到[事件B]的时间线是否连续？
3. [角色名]做这个决定的动机是否在前文中有铺垫？
""")
```

### 自动校验 Prompt

每段写完后，自动执行：

```
ask(question="""
请检查以下段落是否和记忆图谱中的已有信息矛盾：

【待校验段落】
{paragraph_text}

请逐一检查：
1. 角色属性（外貌、性格、能力）是否一致
2. 角色当前状态（受伤、装备、位置）是否和前文衔接
3. 世界观设定（地理、规则）是否合规
4. 时间线是否连续合理
5. 因果关系是否成立
6. 角色是否知道他不应该知道的信息

如果发现矛盾，请指出具体矛盾点和记忆中的原文。
""")
```

## Dream 使用策略

### 每章写完后

```
dream_run(strategy="narrative", seed_count=3)
```

目标：发现本章内容与已有设定的新关联，为后续章节提供灵感。

### 大纲完成后

```
dream_run(strategy="cross_domain", seed_count=5, max_depth=3)
dream_run(strategy="contrastive", seed_count=4)
```

目标：发现设定间隐藏联系，为伏笔设计提供素材。

### 感觉卡文时

```
dream_run(strategy="free_association", seed_count=5)
```

目标：自由联想，打破创作僵局。

### 发现伏笔机会时

```
dream_run(strategy="leap", seed_count=3)
dream_run(strategy="temporal_bridge", seed_count=3)
```

目标：远距离联想、时间线关联，创造意外伏笔。

## 错误处理

| 情况 | 处理方式 |
|------|---------|
| quick_search 无结果 | 降低 similarity_threshold，换关键词重试 |
| remember 抽取的实体不完整 | 用 create_entity 手动补全 |
| ask 返回矛盾信息 | 检查 source_name 是否有重复写入，用 timeline 追溯冲突来源 |
| 角色状态不明 | 用 entity_profile + get_entity_timeline 完整回溯 |
| 伏笔丢失 | 检查 source_name="伏笔-*" 的 episode，重新确认状态 |
