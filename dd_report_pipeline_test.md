
**实体对齐 (S6) 占每窗口 51.6% 的时间, 是绝对瓶颈。**

根本原因:
1. 每个 batch_resolve 实体需 3-5s LLM 调用 (GPU 串行)
2. 后续窗口的已有实体越多, 候选匹配越多, 需 LLM 判断的越多
3. W4 有 88 个新实体对齐 ~155 个已有实体 → 408s

## 七、结论与建议

### 已验证的优化效果

1. **挑战式 Prompt**: 精炼贡献 57% 实体, 每窗口平均 62 实体 + 71 关系
2. **实体内容 Fast-path**: 95.6% 命中, 步骤3 从 ~346s 降至 4.5s (**77x**)
3. **关系内容 Fast-path**: 69.3% 命中, 关系内容从 ~340s 降至 32.4s (**10x**)
4. **对齐 Fast-path**: 73.8% 无需 LLM (no_candidates + exact_match_fast)

### 核心瓶颈: S6 实体对齐

- 占每窗口 51.6% 时间
- 随图谱增长而增长 (W4: 408s / W6: 261s)
- 每个 batch_resolve LLM 调用 3-5s, GPU 串行无法并行

### 下一步建议

1. **S6 批量对齐**: 将多个实体的对齐判断合并到一次 LLM 调用中 (当前是逐个调用)
2. **候选预过滤**: 对低相似度候选提前排除, 减少 LLM 调用数
3. **增量对齐**: 只对新增候选调用 LLM, 缓存已有判断结果
4. **预计总时间**: 16 窗口完整运行约需 60-90 分钟 (当前速率)
